# EV-012 - ERAS MDB Decision Intake

Status: validated-decision-intake

Date: 2026-04-24

## Claim Supported

The ERAS MDB human decision workflow can validate and record a complete human review decision in the versioned decision CSV without reading MDB business rows and without declaring an authoritative database.

## Evidence Sources

- Decision artifact: `docs/reviews/ERAS_MDB_HUMAN_DECISIONS_20260424.csv`
- Decision status report: `docs/reports/eras_mdb_human_decision_status_20260424.md`
- Decision intake implementation: `src/mcp_server/facade.py`, `src/mcp_server/cli.py`, `src/mcp_server/server.py`
- Review service: `src/services/eras_mdb_review.py`
- Tests: `tests/unit/test_eras_mdb_review.py`, `tests/unit/test_mcp_facade_readonly.py`

## Facts Recorded

- New CLI/MCP command: `eras-set-review-decision`
- Dry-run mode validates an intended decision without writing files.
- Write mode is scoped to:
  - `docs/reviews/ERAS_MDB_HUMAN_DECISIONS_20260424.csv`
  - `docs/reports/eras_mdb_human_decision_status_20260424.md`
- `accept_review` and `reject_review` require `reviewer` and `decision_basis`.
- `reviewed_at_utc` is generated by the tool for accepted/rejected decisions.
- Current versioned decision state remains `human_review_pending`.

## Validation

- Dry-run against `CLIENT\client.mdb` returns `human_review_ready` preview and writes no files.
- Unit tests prove complete accepted decisions can be written in a temporary workspace.
- Unit tests prove incomplete accepted decisions and unknown `source_path` values are refused.
- Unit tests prove rejected decisions carry reviewer, UTC date, and decision basis.

## Limits

- No real human decision has been applied to the versioned CSV in this cycle.
- The tool does not promote `human_review_ready` to an authoritative MDB selection.
- No MDB row values or data samples were read.
